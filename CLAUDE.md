## 회사 개요

**크리에이터후드 (Creatorhood)**는 스포츠 라이프스타일 콘텐츠 스튜디오다.

- 타겟: "운동에서 배운 몰입의 에너지를 삶 전반에 적용하는 사람들"
- 2025년까지 인스타그램 매거진 → 2026년부터 유튜브+인스타+웹매거진 영상 중심 콘텐츠 스튜디오로 전환
- 대표: 정수현 (@yoll_sugi)
- 인스타그램: @creatorhood.mag

## 작업 컨벤션

### 언어
- 문서 작성: 한국어 기본 (시리즈명, 브랜드명 등 고유명사는 영문 유지)
- 코드/기술 문서: 한영 혼용

### 파일 네이밍
- 세션 백업: `SESSION_BACKUP_[주제]_[YYMMDD].md`
- 내부 문서: 한글 파일명 사용 가능
- 영문 문서: kebab-case

### 문서 작업 흐름 (md → Notion)
- 내용이 많이 바뀔 수 있는 기획/설계 문서는 **md 파일에서 먼저 작업**하고 버전 관리
- md에서 충분히 얼라인한 뒤, 사용자가 컨펌하면 그때 Notion 페이지 생성
- Notion 페이지를 먼저 만들지 말 것 — md가 선행

### 워크 디렉토리별 역할
각 `claude-*` 디렉토리는 특정 업무 영역의 Claude 작업 공간이다. 디렉토리 안에서 작업할 때 해당 영역의 맥락을 우선한다.

### 세션 종료 시 git 동기화 (필수)
세션이 끝나거나 사용자가 마무리 신호를 보내면 **항상** 아래 순서로 실행:
```
git add -A
git commit --no-verify -m "..."
git push --no-verify origin main
```
- Mac mini ↔ MacBook 동기화 목적
- main 브랜치 직접 커밋/push 시 `--no-verify` 필요

### 주의사항
- PDF 파일이 많음 — 소개서, 포트폴리오, 제안서 등 참고용
- `.skill`, `.plugin` 파일은 Claude 커스텀 스킬/플러그인 정의 파일
- 경영/재무/법률 관련 문서는 Private 저장소(2026-CH-private)에서 관리
