# 카카오 노션 협업 교육 — 실습 템플릿 구성안

> 교육에서 수강생이 직접 구축하게 될 노션 템플릿의 구조와 속성 설계.
> 이 문서를 기반으로 실제 노션 템플릿을 제작하고 캡처본을 준비한다.

---

## 템플릿 전체 구조

교육용 템플릿은 **하나의 팀 워크스페이스 페이지** 안에 모든 요소가 연결된 형태로 구성.
수강생은 이 페이지를 복제(Duplicate)하여 실습을 진행한다.

```
📌 [팀명] 팀 워크스페이스
├── 🏠 팀 홈페이지 (대시보드)
├── 🎯 프로젝트 DB
├── ✅ 태스크(할일) DB
├── 📝 회의록 DB
└── 📚 팀 레퍼런스 아카이브 DB
```

**핵심: 4개 DB가 관계형 속성으로 서로 연결됨**

```
프로젝트 DB ←→ 태스크 DB
     ↕              ↕
회의록 DB    팀 레퍼런스 DB
```

## 템플릿 1: 팀 홈페이지 (대시보드)

> 커리큘럼 모듈 3-1 에서 활용

### 페이지 구성

| 구역 | 내용 | 노션 블록 |
|------|------|----------|
| 헤더 | 팀명, 팀 미션/비전 한줄 | 제목 + Callout |
| 팀원 디렉토리 | 멤버 이름, 역할, 연락처 | 테이블 (심플 테이블 or 인라인 DB) |
| 공지사항 | 팀 공지, 중요 안내 | 토글 리스트 |
| 바로가기 | 프로젝트DB, 회의록DB, 아카이브 등 링크 | 컬럼 + 링크드 뷰 |
| 진행 중 프로젝트 | 프로젝트 DB의 "진행 중" 필터 뷰 | 링크드 뷰 (보드) |
| 이번 주 할일 | 태스크 DB의 "이번 주" 필터 뷰 | 링크드 뷰 (리스트) |

### 레이아웃 (2컬럼)

```
┌─────────────────────────────────────────┐
│  🏠 [팀명] 팀 홈페이지                      │
│  "우리 팀의 미션을 한 문장으로"                 │
├────────────────────┬────────────────────┤
│  📋 공지사항          │  👥 팀원 디렉토리      │
│  ▸ [3/10] 주간회의..  │  수현 | PD | @...    │
│  ▸ [3/8] 신규 프로...  │  지운 | 에디터 | @... │
├────────────────────┴────────────────────┤
│  🎯 진행 중 프로젝트 (보드 뷰)                │
│  ┌──────┐ ┌──────┐ ┌──────┐            │
│  │준비중  │ │진행중  │ │완료   │            │
│  │ ...  │ │ ...  │ │ ...  │            │
│  └──────┘ └──────┘ └──────┘            │
├─────────────────────────────────────────┤
│  ✅ 이번 주 할일 (리스트 뷰)                  │
│  □ 콘텐츠 기획안 초안 작성 | 수현 | 3/12      │
│  □ 디자인 시안 피드백 | 우석 | 3/11          │
└─────────────────────────────────────────┘
```

### 기존 모듈 활용
- **[팀페이지 템플릿] 스프린트/프로젝트/태스크/회의록 연동하기** 페이지의 컬럼 레이아웃 참고
- 팀 홈페이지 상단 구성은 **Anthropic 프로젝트 플래너** 스타일 참고하여 깔끔하게

## 템플릿 2: 프로젝트 DB

> 커리큘럼 모듈 3-2 에서 활용

### DB 속성 설계

| 속성명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| 프로젝트명 | Title | 프로젝트 이름 | "2026 Q1 마케팅 캠페인" |
| 상태 | Status | 프로젝트 진행 상태 | 준비중 → 진행중 → 완료 |
| 담당자 | Person | 프로젝트 리드 | @수현 |
| 시작일 | Date | 프로젝트 시작 | 2026-03-01 |
| 마감일 | Date | 프로젝트 마감 | 2026-03-31 |
| 우선순위 | Select | 중요도 | 🔴 높음 / 🟡 보통 / 🟢 낮음 |
| 태스크 | Relation → 태스크 DB | 연결된 할일 목록 | (자동 연결) |
| 회의록 | Relation → 회의록 DB | 관련 회의록 | (자동 연결) |
| 진행률 | Rollup (태스크 완료%) | 태스크 완료 비율 자동 계산 | 75% |

### 뷰 구성

| 뷰 이름 | 뷰 타입 | 필터/정렬 | 용도 |
|---------|--------|----------|------|
| 전체 프로젝트 | 테이블 | 상태별 정렬 | 전체 조회 |
| 보드 | 보드 (상태 기준) | - | 진행 상황 한눈에 |
| 타임라인 | 타임라인 | 시작일~마감일 | 일정 관리 |
| 내 프로젝트 | 테이블 | 담당자=나 | 개인 뷰 |

### 페이지 템플릿 (프로젝트 상세)
```
📋 프로젝트 개요
  - 목표:
  - 핵심 지표:
  - 참여 멤버:

📎 관련 태스크 (링크드 뷰 — 태스크 DB 필터)
📝 관련 회의록 (링크드 뷰 — 회의록 DB 필터)
📌 메모 & 참고자료
```

## 템플릿 3: 태스크(할일) DB

> 커리큘럼 모듈 3-2 에서 프로젝트 DB와 함께 활용

### DB 속성 설계

| 속성명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| 할일 | Title | 태스크 이름 | "키비주얼 시안 제작" |
| 상태 | Status | 진행 상태 | 할일 → 진행중 → 완료 |
| 담당자 | Person | 실행 담당자 | @우석 |
| 마감일 | Date | 태스크 마감 | 2026-03-15 |
| 우선순위 | Select | 중요도 | 🔴 / 🟡 / 🟢 |
| 프로젝트 | Relation → 프로젝트 DB | 소속 프로젝트 | "Q1 마케팅 캠페인" |
| 태그 | Multi-select | 업무 분류 | 디자인, 기획, 개발, 리서치 |

### 뷰 구성

| 뷰 이름 | 뷰 타입 | 필터/정렬 | 용도 |
|---------|--------|----------|------|
| 보드 (상태별) | 보드 | 상태 그룹 | 칸반 보드 |
| 내 할일 | 리스트 | 담당자=나, 완료 제외 | 개인 업무 |
| 이번 주 | 테이블 | 마감일=이번 주 | 주간 관리 |
| 캘린더 | 캘린더 | 마감일 기준 | 일정 확인 |

## 템플릿 4: 회의록 DB

> 커리큘럼 모듈 3-3 에서 활용

### DB 속성 설계

| 속성명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| 회의명 | Title | 회의 제목 | "3/10 주간 정기회의" |
| 회의 유형 | Select | 회의 분류 | 정기회의 / 프로젝트 / 브레인스토밍 / 1on1 |
| 일시 | Date | 회의 날짜·시간 | 2026-03-10 10:00 |
| 참석자 | Person | 참석 인원 | @수현, @지운 |
| 프로젝트 | Relation → 프로젝트 DB | 관련 프로젝트 | "Q1 마케팅 캠페인" |
| 상태 | Status | 회의록 작성 상태 | 예정 → 작성중 → 완료 |

### 페이지 템플릿 (회의록 본문)
```
📋 회의 정보
  - 일시:
  - 참석자:
  - 안건:

💬 논의 내용
  1.
  2.

✅ 실행 항목 (Action Items)
  - [ ] [담당자] 내용 | 마감일
  - [ ] [담당자] 내용 | 마감일

📎 참고 자료
```

### 뷰 구성

| 뷰 이름 | 뷰 타입 | 필터/정렬 | 용도 |
|---------|--------|----------|------|
| 최근 회의 | 리스트 | 일시 내림차순 | 기본 뷰 |
| 캘린더 | 캘린더 | 일시 기준 | 일정 조회 |
| 프로젝트별 | 테이블 | 프로젝트 그룹핑 | 프로젝트 맥락 확인 |

## 템플릿 5: 팀 레퍼런스 아카이브 DB

> 커리큘럼 모듈 4 에서 활용

### DB 속성 설계

| 속성명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| 제목 | Title | 자료 제목 | "2026 마케팅 트렌드 리포트" |
| URL | URL | 원본 링크 | https://... |
| 카테고리 | Select | 자료 분류 | 마케팅 / 디자인 / 기술 / 경영 / 기타 |
| 태그 | Multi-select | 세부 태그 | SNS, 트렌드, 벤치마킹 |
| 등록자 | Person | 누가 등록했는지 | @수현 |
| 등록일 | Created time | 자동 기록 | (자동) |
| 메모 | Text | 핵심 요약, 코멘트 | "Q1 캠페인 기획 시 참고" |
| 프로젝트 | Relation → 프로젝트 DB | 관련 프로젝트 연결 (선택) | "Q1 마케팅 캠페인" |

### 뷰 구성

| 뷰 이름 | 뷰 타입 | 필터/정렬 | 용도 |
|---------|--------|----------|------|
| 전체 | 갤러리 | 등록일 내림차순 | 카드형 브라우징 |
| 카테고리별 | 테이블 | 카테고리 그룹핑 | 분류별 조회 |
| 내가 등록한 | 테이블 | 등록자=나 | 내 자료 관리 |

### 기존 모듈 활용
- **모두의연구소 "숲을 볼 수 있는 똑똑한 지식관리법"** 강의의 레퍼런스 아카이브 구조 참고
- **PARA 프레임워크** (Projects/Areas/Resources/Archive) 개념을 간소화하여 적용
- **노션 웹 클리퍼** 연동 — 웹에서 바로 이 DB에 자료 저장하는 실습 포함

## 관계형 연동 맵 (핵심)

```
┌──────────────┐     Relation      ┌──────────────┐
│  프로젝트 DB   │ ←──────────────→ │   태스크 DB    │
│              │                   │              │
│  Rollup:     │                   │              │
│  태스크 완료%  │                   │              │
└──────┬───────┘                   └──────────────┘
       │
       │ Relation
       │
┌──────┴───────┐     Relation      ┌──────────────┐
│  회의록 DB    │                   │ 레퍼런스 DB    │
│              │                   │              │
│              │                   │  Relation →   │
│              │                   │  프로젝트 DB   │
└──────────────┘                   └──────────────┘
```

**수강생이 직접 만드는 관계형 연동**:
1. 태스크 DB → 프로젝트 DB (어떤 프로젝트의 할일인지)
2. 회의록 DB → 프로젝트 DB (어떤 프로젝트에 대한 회의인지)
3. 레퍼런스 DB → 프로젝트 DB (어떤 프로젝트에 관련된 자료인지)
4. 프로젝트 DB에 Rollup 추가 (태스크 완료율 자동 계산)

## 캡처 필요 항목 (탈잉 전달용)

> 수현이 실제 노션에서 템플릿을 제작한 뒤 아래 항목을 캡처하여 전달

| # | 캡처 대상 | 설명 |
|---|----------|------|
| 1 | 팀 홈페이지 전체 | 대시보드 레이아웃이 보이는 전체 캡처 |
| 2 | 프로젝트 DB 보드 뷰 | 상태별 프로젝트 카드가 보이는 보드 |
| 3 | 프로젝트 상세 페이지 | 관계형으로 태스크와 회의록이 연결된 모습 |
| 4 | 태스크 DB 칸반 보드 | 할일 → 진행중 → 완료 흐름 |
| 5 | 회의록 페이지 + 실행항목 | 회의록 본문 템플릿과 체크리스트 |
| 6 | 레퍼런스 아카이브 갤러리 뷰 | 카드형으로 자료가 정리된 모습 |
| 7 | 관계형 연동 시연 | 태스크에서 프로젝트를 연결하는 모습 |
