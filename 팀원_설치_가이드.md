# 팀원 설치 가이드

> Git 저장소 clone부터 플러그인 설치까지, 처음부터 따라하는 가이드
> 대상: 김지운, 임우석, 임용

---

## 순서 요약

```
Step 1  터미널 열기
Step 2  Git 설치 확인
Step 3  저장소 clone (파일 다운로드)
Step 4  Claude Cowork에서 플러그인 설치
Step 5  커넥터 연결 (Notion, Slack)
Step 6  테스트
Step 7  업데이트 받기 (이후 반복)
```

---

## Step 1. 터미널 열기

Mac에서 **터미널**을 연다.

**여는 방법 (택 1)**:
- `Cmd + Space` → "터미널" 입력 → Enter
- Finder → 응용 프로그램 → 유틸리티 → 터미널

터미널이 열리면 아래와 같은 화면이 나온다:

```
사용자이름@MacBook ~ %
```

이게 보이면 준비 완료.

---

## Step 2. Git 설치 확인

터미널에 아래를 복사해서 붙여넣고 Enter:

```bash
git --version
```

**결과가 이렇게 나오면** → 이미 설치됨, Step 3으로 이동:
```
git version 2.xx.x
```

**"command not found"가 나오면** → 설치 필요:
```bash
xcode-select --install
```
팝업이 뜨면 "설치" 클릭. 설치 후 터미널을 껐다 다시 열고 `git --version` 재확인.

---

## Step 3. 저장소 clone (파일 다운로드)

터미널에 아래를 복사해서 붙여넣고 Enter:

```bash
cd ~/Desktop && git clone https://github.com/Yollsugi1/2026-CH.git
```

**무슨 일이 일어나나**: 바탕화면에 `2026-CH` 폴더가 생긴다. 이 안에 팀 공유 문서, 스킬, 플러그인이 모두 들어있다.

**확인 방법**: 바탕화면에 `2026-CH` 폴더가 보이면 성공.

### 폴더 안 구조

```
바탕화면/2026-CH/
├── CLAUDE.md                          ← 팀 공유 맥락 (자동 로드)
├── 크리에이터후드_팀_Claude_활용_가이드.md  ← 전체 활용 가이드
├── claude-cowork-contents/            ← 콘텐츠 스킬/플러그인
│   ├── ch-mag-writer.plugin           ← 임용 전용
│   └── ...
├── 국순당-자동화-팀공유/               ← 국순당 스킬/플러그인
│   ├── kuksundang-workflow.plugin     ← 우석 전용
│   ├── 설치-가이드.md
│   ├── 임우석-Plugin-사용가이드.md
│   └── skills/
└── claude-team-zonex/                 ← Zone X 프로젝트
```

## Step 4. Claude Cowork에서 플러그인 설치

### 4-1. Claude 앱 실행

Claude Desktop 앱을 열고 **Cowork 모드**로 진입한다.
(아직 Claude 앱이 없으면 https://claude.ai/download 에서 다운로드)

### 4-2. 본인 역할에 맞는 플러그인 설치

#### 우석 (디자이너 / 국순당 담당)

1. Finder에서 바탕화면 → `2026-CH` → `국순당-자동화-팀공유` 폴더 열기
2. **`kuksundang-workflow.plugin`** 파일을 찾는다
3. 이 파일을 **Claude Cowork 대화창에 드래그앤드롭**
4. 플러그인 미리보기가 나타나면 **"수락(Accept)"** 클릭

설치 후 사용할 수 있는 커맨드:

| 커맨드 | 기능 | 자연어로도 가능 |
|---|---|---|
| `/daily` | 데일리 리포트 | "오늘 현황 보내줘" |
| `/schedule` | 일정 확인 | "다음 공유 언제?" |
| `/weekly` | 주간 리포트 | "이번 주 현황" |
| `/client-update` | 클라이언트 보고 | "연정님 보고 만들어줘" |
| `/plan [주제]` | 콘텐츠 기획 | "/plan 순당포착 영동시장" |

**상세 사용법**: `국순당-자동화-팀공유/임우석-Plugin-사용가이드.md` 참고

#### 임용 (에디터 / CH MAG 담당)

1. Finder에서 바탕화면 → `2026-CH` → `claude-cowork-contents` 폴더 열기
2. **`ch-mag-writer.plugin`** 파일을 찾는다
3. 이 파일을 **Claude Cowork 대화창에 드래그앤드롭**
4. **"수락(Accept)"** 클릭

설치 후 사용할 수 있는 워크플로우:

| 단계 | 커맨드 | 하는 일 |
|---|---|---|
| Phase 1 | "소재 접수: [주제]" | 소재 분석 & 소싱 확장 |
| Phase 2 | "앵글 잡아줘" | 구조 기획 (반복 수정) |
| Phase 3 | "Draft 써줘" | 글 초안 작성 |
| Phase 4 | "피드백 반영: [내용]" | 수정 & 버전 기록 |
| Phase 5 | "인스타로 변환" | Instagram 캡션/카드뉴스 변환 |

**새 세션 시작할 때**: `CH-MAG-WRITER-정임용-Context-Starter.md` 파일의 내용을 복사해서 Claude 대화에 붙여넣으면 이전 작업 맥락을 이어갈 수 있다.

#### 김지운 (콘텐츠 PD)

1. 우석과 동일하게 **`kuksundang-workflow.plugin`** 설치
2. 추가로, `claude-cowork-contents` 폴더의 문서들을 참고하면 콘텐츠 현황 파악에 유용

주로 쓸 기능:

```
"이번 주 촬영 일정 정리해줘"
"순당포착 4화 마감이 언제야?"
"/schedule"
```

## Step 5. 커넥터 연결 (Notion, Slack)

플러그인이 Notion/Slack과 연동되려면 커넥터를 연결해야 한다.

### Notion 연결

1. Claude Cowork → 설정(Settings) → Connectors
2. **Notion** 찾아서 **연결(Connect)** 클릭
3. Notion 로그인 → 액세스 허용

또는 Claude에게 직접 말해도 된다:
```
"Notion 연결해줘"
```

### Slack 연결

1. Claude Cowork → 설정(Settings) → Connectors
2. **Slack** 찾아서 **연결(Connect)** 클릭
3. Slack 로그인 → 워크스페이스 선택 → 허용

또는:
```
"Slack 연결해줘"
```

### 연결 확인

Claude에게 이렇게 물어본다:
```
"현재 연결된 커넥터 보여줘"
```
Notion과 Slack이 모두 나오면 완료.

---

## Step 6. 테스트

설치가 끝났으면 간단히 테스트한다.

### 우석 / 지운 테스트

Claude Cowork에서:
```
국순당 오늘 할 일 정리해줘
```
→ Notion에서 데이터를 가져와서 오늘의 할 일 목록이 나오면 성공.

### 임용 테스트

Claude Cowork에서:
```
CH MAG 스킬 세팅 확인해줘
```
→ 설치된 스킬과 레퍼런스 파일 목록이 나오면 성공.

### 안 되면?

| 증상 | 해결 |
|---|---|
| 커맨드가 안 먹힘 | 플러그인 재설치 (드래그앤드롭 다시) |
| "Notion 조회 실패" | 커넥터 재연결 (설정 → Connectors) |
| "Slack 발송 실패" | Slack 커넥터 권한을 "항상 허용"으로 변경 |
| 데이터가 안 나옴 | Notion DB에 날짜 필드가 비어있지 않은지 확인 |

그래도 안 되면 수현 대표에게 연락.

---

## Step 7. 업데이트 받기 (이후 반복)

수현 대표가 스킬을 수정하거나 새 문서를 추가하면, 아래 명령어로 최신 버전을 받는다.

터미널에서:

```bash
cd ~/Desktop/2026-CH && git pull
```

**이것만 기억하면 된다: `git pull` = 최신 파일 받기**

업데이트 후 플러그인이 변경되었다면, Step 4의 드래그앤드롭을 다시 한 번 해주면 된다.
