# Session Backup

세션의 전체 맥락을 구조화된 마크다운으로 기록합니다. 핵심 원칙은 **요약하지 않는 것** — 대화의 흐름과 의사결정 과정, 단계별 산출물을 있는 그대로 보존합니다.

## 왜 요약이 아닌 전체 기록인가

요약은 편리하지만 맥락을 잃는다. 새 세션에서 "왜 이 결정을 했는지", "중간에 어떤 변경이 있었는지"를 모르면 작업을 제대로 이어갈 수 없다. 특히 클라이언트 미팅 전후로 조건이 바뀌거나, 여러 번의 수정을 거친 프로젝트는 그 변경 과정 자체가 중요한 맥락이다.

## 백업 실행 절차

### Step 1: 대화 흐름 전체 재구성

세션에서 일어난 모든 일을 **시간 순서대로** 기록한다. 각 단계마다:
- 사용자가 무엇을 요청했는지 (원문에 가깝게)
- Claude가 무엇을 했는지 (어떤 도구를 썼고, 어떤 파일을 만들었는지)
- 결과물이 무엇이었는지 (파일명, 핵심 내용)
- 중간에 변경/수정이 있었다면 무엇이 왜 바뀌었는지

### Step 2: 구조화된 백업 파일 작성

파일명: `SESSION_BACKUP_{프로젝트명}_{YYMMDD}.md`

다음 구조를 따른다:

```markdown
# 세션 백업 — {프로젝트명}

**백업 일자**: YYYY-MM-DD
**프로젝트 상태**: {현재 단계 한 줄 요약}

---

## 🔑 핵심 사항
현재 확정된 의사결정 사항만 간결하게.
새 세션에서 가장 먼저 파악해야 할 정보.

## 📝 전체 대화 흐름 (시간순)

### [1] {첫 번째 작업 — 제목}
**사용자 요청**: "{사용자가 말한 내용 — 원문에 가깝게 기록}"

**진행 내용**:
- {Claude가 한 일 상세 기술}
- {사용한 도구, 참조한 파일}
- {중간 판단이나 의사결정}

**산출물**:
- `파일명.ext` — {파일 설명, 핵심 내용 요약}
- {노션/외부 링크가 있다면 URL 포함}

**결정된 사항**:
- {이 단계에서 확정된 것}

---

### [2] {두 번째 작업 — 제목}
**사용자 요청**: "..."
...
(동일 구조 반복)

---

### [N] {마지막 작업}
...

## 📁 최종 파일 목록
| 파일명 | 위치 | 설명 | 버전 |
|---|---|---|---|
| file.ext | workspace root | 설명 | 최신/구버전 |

## 🔗 외부 링크
- 노션, 구글드라이브 등

## ⏭️ 다음 할 일
- [ ] 미완료 작업
- [ ] 후속 작업

## 💡 맥락 메모
- 클라이언트/상대방 성향 및 선호
- 기술적 이슈/워크어라운드 (MCP 에러 등)
- 참고할 스킬이나 레퍼런스 파일
- 향후 세션에서 주의할 점

## 🔄 이 백업 사용법
새 세션에서:
{프로젝트명} 프로젝트 이어서 진행하려고 해.
SESSION_BACKUP_{파일명}.md 파일 읽고 맥락 파악한 다음 진행해줘.
```

### Step 3: 워크스페이스에 저장

백업 파일을 현재 워크스페이스 폴더 루트에 저장한다.

## 작성 원칙

1. **요약 금지**: 대화를 줄여서 정리하지 않는다. 흐름을 있는 그대로 기록한다.
2. **원문 보존**: 사용자 요청은 원문에 가깝게 인용한다. 의도를 재해석하지 않는다.
3. **변경 이력 필수**: "처음에는 A였다가 B로 바뀌었다"는 과정을 반드시 기록한다.
4. **숫자 정확**: 금액, 수량, 일정 등 핵심 숫자는 빠짐없이, 정확하게 기록한다.
5. **파일 추적**: 생성된 모든 파일의 정확한 경로, 용도, 버전(최신/구버전) 기록한다.
6. **기술적 이슈**: MCP 에러, 워크어라운드 등 반복될 수 있는 기술 이슈를 기록한다.
7. **Self-contained**: 이 파일 하나만 읽으면 맥락이 100% 복원되어야 한다.

## 여러 프로젝트 관리

- 프로젝트별로 별도 백업 파일 생성
- 같은 프로젝트의 이전 백업은 덮어쓰지 않고 날짜로 구분
- 예: `SESSION_BACKUP_신상윤_250215.md`, `SESSION_BACKUP_신상윤_250218.md`

## 프로액티브 사용

세션이 길어지면 (대략 10턴 이상의 작업 후) 사용자에게 먼저 제안:
> "작업이 상당히 진행됐는데, 세션 백업해둘까요? 다음에 이어서 하기 편하게요."
