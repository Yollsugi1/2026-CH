# DORM 채널 실행플랜

**채널**: DORM (@dorm.mag 가칭)
**작성일**: 2026-03-07
**채널 리더**: 은서 (총괄) / 이선 (초기 구조 세팅, 이후 IF 전환)
**에디터 풀**: 예영 관리 (대학생 2-3명)
**런칭 목표**: 2026년 4월 중순

---

## 채널 정체성

**슬로건**: "스포츠에 잠들어 있던 이야기들 — DORM"

**포지셔닝**: 타 인스타 매거진이 소식을 전달할 때, DORM은 그 소식 속 팬의 감정을 아카이빙한다.
- 정보 X → 사람의 경험과 감정 O
- 대학생 에디터 = 실제 팬 → 진정성 온도
- 빠른 제작 파이프라인 → 소식에 즉시 반응

**CH MAG과의 관계**: CH MAG(well-edited, 공수 높음)의 라이트 버전. 같은 스포츠 라이프스타일 씬을 더 빠르고, 더 가볍게, 사람 중심으로 다룬다.

## 콘텐츠 구조

### 레이어 1 — 상시 기반 콘텐츠 (매일 2-3개 생산 가능한 구조)

스포츠 라이프스타일 씬의 단편 소재들. 특정 시리즈명 없이 상시 올라가는 기반 콘텐츠.

**소재 유형**:
- 이번 주 화제 KBO/NBA/스니커즈/피트니스 소식 → 그 소식 속 사람 앵글
- 트렌드 아이템을 실제로 산 사람 이야기
- 직관 갔다 온 사람의 하루
- 스포츠 팬덤 주변 일상 소재 (시즌권 추첨 실패, 구단 유니폼 도착 언박싱 등)

**제작 원칙**:
- 편당 1-2시간 이내 완결 (상시 기반이므로 시리즈보다 빠르게)
- 스마트폰 촬영 OK, 현장감 우선
- 에디터가 직접 경험하거나 주변 취재한 소재 우선
- 캡션: 3줄 이내, 대화체

**포맷 믹스** (매일 2-3개 = 다포맷 동시 활용):
- 릴스 30-60초
- 캐러셀 3-5장 (정보 밀도 낮음, 사진 중심)
- 스토리 (결과 공유, 투표 활용)

> **Figma 연동 예정**: 상시 기반 콘텐츠의 그래픽 포맷을 Figma 템플릿으로 표준화하여 에디터가 직접 디자인 작업 가능하게 세팅할 계획. 추후 Claude Code ↔ Figma 연동.

---

### 레이어 2 — 정기 시리즈 (주 1-2회, 이선 아이데이션 기반)

기반 콘텐츠 사이에 정기적으로 올라가는 시리즈. 채널 브랜딩과 아카이빙 자산 역할.

#### 시리즈 A — Fan Archive
**원안**: 이선의 "Why We Love"
**컨셉**: 스포츠를 좋아하는 이유를 인터뷰로 꺼낸다. 정보가 아니라 인생 이야기처럼 풀기.

**고정 질문 구조**:
1. 왜 이 팀(종목)을 좋아하세요?
2. 가장 울었던 경기는?
3. 지금까지 본 경기 중 가장 기억에 남는 장면은?
4. 스포츠는 당신에게 뭔가요? (항상 마지막)

**예시 타이틀**:
- "이 사람은 몇 년째 8번 유니폼만 산다"
- "이 사람은 4종 스포츠 팬이다"
- "최다 야구장 방문 나야나"
- "짜증나도 몇 년째 시즌권 구매 중"

**포맷**: 릴스 60-90초 + 캐러셀 요약
**빈도**: 주 1회

---

#### 시리즈 B — Fan's POV
**원안**: 이선의 "Fan's POV"
**컨셉**: 경기 분석 X, 팬의 감정을 날것으로 기록 O.

**소재 유형**:
- 직관 전/후 표정 변화 (몇 회마다 달라지는 표정)
- 이적 기사 뜬 날의 감정 기록
- 시즌 시작 기대 VS 시즌 끝 현실
- 러너 초보의 달리기 표정 변화
- "헬스 주 5일 선언" 후 실제 결과

**포맷**: 릴스 60초 (현장감 최우선, 편집 최소화)
**빈도**: 주 1-2회

---

#### 시리즈 C — 팬 소비 리포트
**원안**: 이선의 "스포츠 취향 분석실"
**컨셉**: 팬 대변인. 스포츠 소비 데이터를 솔직하게 공개. 마케터/브랜드가 팔로우할 이유가 있는 콘텐츠.

**소재 유형**:
- "직관 가면 이렇게 돈 써요…"
- "우리 구단 새 유니폼 리뷰"
- "구단별 티켓팅 현황 보여드림"
- "저 필라테스 시작해요 — 얼마 드냐면요"
- "매일 단백질 쉐이크 (할인 좀 !!!)"

**포맷**: 캐러셀 저장형 (정보 밀도 적당, 저장 유발)
**빈도**: 주 1회

## 콘텐츠 파이프라인

### 주간 흐름

```
월요일: 은서 — 이번 주 소재 확정 + 에디터 배정
        수현 — 주간 스포츠 씬 힌트 공유 (선택)

화~목: 에디터 — 취재·촬영·편집 (편당 1-2h)
        은서 — 상시 기반 콘텐츠 1-2개 직접 제작 또는 확인

금요일: 임용 — 카피 교정 + 사실 확인 (주 1-2h)
         은서 — 다음 주 예약 발행 설정
```

### 콘텐츠 캘린더 구조

| 구분 | 월 | 화 | 수 | 목 | 금 | 토 | 일 |
|---|---|---|---|---|---|---|---|
| 상시 기반 | 1 | 1 | 1 | 1 | 1 | — | — |
| 시리즈 | — | — | A or B | — | C | — | — |

→ 평일 주 6-7개. 주말은 비정기 or 없음.
→ 상시 기반 콘텐츠 매일 1개 + 시리즈 주 2-3개 = 주 7-9개 목표 (안정화 후)
→ **초기(런칭~4주)**: 주 3-4개로 시작, 에디터 온보딩 후 점진적 확대

---

## 에디터 운영 구조

### 에디터 역할

취재자·스토리텔러. 스킬보다 커뮤니케이션 능력과 관심 우선.

- 주간 소재 힌트 받음 → 사람 찾아 섭외 → 현장 취재·촬영 → 원고 초안 제출
- 편당 1-2시간 완결이 목표
- 스마트폰 촬영 기본. 특별한 장비 없음.

### 소재 파이프라인

```
[수현] 월 1회 방향 + 수시 힌트 → [은서] 주간 소재 확정 → 에디터 배정
→ [에디터] 취재·편집 → [임용] 카피 교정 → 발행
```

### 채널 리더 전환 기준

이선 → 은서 단독 운영 전환 (3가지 조건 충족 시):
1. 주 3-4개 업로드 루틴 2주 연속 유지
2. 에디터 2-3명 자립적 취재 가능
3. 은서가 단독 운영 가능하다고 확인

## 자동화 방안

| 자동화 대상 | 방법 |
|---|---|
| 주간 소재 수집 | Claude 프롬프트로 스포츠·라이프스타일 씬 주간 이슈 정리 (에디터 취재 힌트용) |
| 캡션 초안 | 에디터 취재 요약 입력 → Claude 훅+본문+CTA 생성 |
| 업로드 스케줄 | Notion DB 주간 발행 일정 자동 생성 |
| 그래픽 템플릿 | Figma 표준 템플릿 (추후 세팅) |

**소재 수집 Claude 프롬프트**:
```
이번 주 스포츠·라이프스타일 씬에서 "사람의 이야기"로 다룰 수 있는 소재 5개 제안.
조건: 단순 뉴스가 아닌, 팬·소비자·플레이어 시선으로 접근 가능한 것.
KBO, NBA, 스니커즈, 피트니스 트렌드에서 골고루. 각 소재는 3줄 이내.
```
