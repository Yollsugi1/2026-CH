## 콘텐츠 구조

### 메인 시리즈 — IF

**컨셉**: "만약에 ~라면?" 가설 질문을 던진다. 질문 자체가 대화 소재가 된다.

**질문 카테고리 (이선 아이데이션 기반)**:
- 관계/상황: "잠수 이별 VS 카톡 이별" / "전애인이랑 같은 조 — 한다 VS 드랍한다"
- IF 과몰입 토크: "WBC 국가대표 된 나 — 등장곡은?" / "타석 루틴은?"
- 현실 딜레마: "정규직 전환 1개월 앞, 인턴 동기가 어려운 상황이면?" / "이상형인데 유사과학 믿으면?"
- 소비: "평생 한 브랜드 운동화만 신어야 한다면?" / "평생 한 나라 음식만 먹어야 한다면?"
- 여행/미래: "혼자 2주 여행이라면 어디?"

**발행 포맷 — 3종 세트** (1개 소재 = 3종 동시 발행):
1. 캐러셀/릴스 — 메인 콘텐츠 (질문 시각화)
2. 스토리 — 해당 IF 질문 투표·퀴즈
3. 캐러셀 마지막 슬라이드 CTA — "여러분이면? 댓글로"

**포맷 상세**:
- 캐러셀: 표지(질문) + 2-3장(상황 설명 or 선택지) + 마지막(CTA)
- 릴스: 60초, 에디터가 직접 IF 질문으로 대화해본 썰
- 스토리: 투표/퀴즈/슬라이더 (24h 인터랙션)

### 보조 시리즈

**썰 공유**: 에디터가 실제 IF 질문으로 대화해본 후기 릴스 (60-90초)
**이 질문 받아봤어?**: 팔로워 답변 공유 캐러셀 (댓글 참여 → 다음 콘텐츠 순환)

## 콘텐츠 파이프라인

```
[수현+임용] 월 1회 IF 질문 30개 생성 (Claude 활용, 2h)
    ↓
[이선] 주간 콘텐츠 큐 구성 + 발행 일정 (주 10-15h)
    ↓
[에디터 2-3명] 질문 시각화 + 썰 공유 영상 제작
    ↓
[임용] 카피 최종 확인 + 스토리 인터랙션 기획 (주 1h)
    ↓
발행 (캐러셀/릴스) + 스토리 3종 세트 병행
    ↓
[이선] 댓글·DM 반응 수집 → 다음 콘텐츠 소재 순환
```

## 자동화 방안

### IF 질문 생성 (핵심 자동화)

```
Claude 월 1회 프롬프트:
"20대 대학생이 처음 만난 사람에게 할 수 있는 IF 질문 30개.
조건:
- 특정 지식 없어도 답할 수 있음
- 답변이 그 사람을 알 수 있게 함
- 너무 무겁지 않고, 1-2문장으로 답 가능
카테고리: 관계/직업·미래/소비/여행/스포츠/음식 각 5개씩"
```

### 멀티포맷 자동 변환

```
IF 질문 1개
→ 캐러셀 스크립트 (Claude)
→ 릴스 스크립트 (Claude)
→ 틱톡 텍스트 오버레이 형식 (Claude)
→ 스토리 투표 문항 (Claude)
= 1개 소재 → 4종 포맷 동시 제작
```

### 반응 분석

- 스토리 투표·퀴즈 결과 → Claude 인사이트 요약
- 반응 좋은 카테고리 다음 달 질문 기획에 반영

## 운영 특성

- **에디터 역할**: 제작자보다 기획자. 질문 직접 써봄 → 리포트 공유.
- **매주 에디터 리포트**: "이 IF 질문을 실제로 친구들한테 써봤는지, 반응은?"
- **반응 좋은 답변**: 팔로워 참여 콘텐츠로 순환 (데이터 활용)
