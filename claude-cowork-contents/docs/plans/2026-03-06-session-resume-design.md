# session-resume 스킬 디자인

**날짜**: 2026-03-06
**상태**: 승인됨

---

## 개요

세션 컨텍스트를 저장하고 복구하는 단일 스킬. 두 가지 모드(Save / Resume)를 가진다.
계정 전환, 세션 종료, 일반 재개 등 모든 상황에서 작업 흐름을 이어갈 수 있게 한다.

---

## 스킬 구조

- **스킬명**: `session-resume`
- **파일**: `skills-export/session-resume/SKILL.md`
- **모드**: Save / Resume

---

## Save 모드

### 트리거
- "세션 저장", "여기서 끊을게", "계정 전환할게", "오늘 여기까지", `/session-save`
- 매 세션 마무리 시 루틴 실행
- 계정 전환 직전 더 상세하게 실행

### 동작
Claude가 현재 세션을 분석하여 타임스탬프 파일 생성:

**저장 위치**: `session-resumes/RESUME_YYMMDD_HHMM.md`

**파일 구조**:
```
# 컨텍스트 핸드오프 — [날짜 시간]

## 현재 작업
- 진행 중인 태스크 (어느 단계까지 왔는지)

## 작업 상태
- ✅ 완료된 것
- 🔄 진행 중
- ⏸ 미완료 / 블로킹 이슈

## 주요 파일 / 경로
- 관련 파일 목록

## 이번 세션 결정 사항
- 확정된 방향/결정들

## 대화 흐름 요약
- 주요 논의 맥락 요약

## 다음 스텝
1. 당장 해야 할 것
2. 그 다음
```

---

## Resume 모드

### 트리거
- `/resume`, "이어서 해줘", "어디까지 했지", "컨텍스트 복구해줘"

### 동작
1. `session-resumes/` 디렉토리의 파일 목록을 날짜+작업내용 요약과 함께 나열
2. 사용자가 번호로 선택
3. 선택된 파일을 읽어 구조화된 브리핑 제시
4. 첫 번째 액션 제안

**목록 예시**:
```
저장된 세션:
1. 2026-03-06 14:30 — session-resume 스킬 디자인 작업
2. 2026-03-05 18:20 — short-caption 스킬 제작
3. 2026-03-04 11:00 — CEO Staff 시스템 구축

몇 번 세션을 불러올까요?
```

---

## 파일 관리

- `session-resumes/` 폴더는 Git 추적 → 계정 전환 후에도 접근 가능
- 파일은 누적 저장 (덮어쓰지 않음)
- 오래된 파일은 수동으로 정리

---

## 사용 흐름

```
[세션 종료 전]
수현: "오늘 여기까지"
Claude: RESUME_260306_1430.md 생성 → "저장했습니다. 다음에 /resume로 이어받으세요."

[새 세션 / 계정 전환 후]
수현: "/resume"
Claude: 저장된 세션 목록 나열
수현: "1"
Claude: 컨텍스트 브리핑 + 첫 액션 제안
```
