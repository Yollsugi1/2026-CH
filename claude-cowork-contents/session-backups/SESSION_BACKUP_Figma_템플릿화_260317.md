---

## 작업 요약

'이미지 대지' section을 참고하여 '편집 대지' section의 40개 프레임(1080×1440, 인스타 4:5)을 분석하고, 글자 깨짐/레이아웃 문제를 해결하기 위한 **Auto Layout 기반 템플릿 4종**을 생성했다.

---

## 발견한 문제점

1. **텍스트가 줄별로 개별 TEXT 노드로 분리됨** — 한 문단이 7~8개 개별 텍스트로 쪼개져 있어 글자 깨짐/밀림 발생
2. **폰트 혼용**: Pretendard ExtraBold, Pretendard Variable ExtraBold, Inter Bold/Regular, Apple SD Gothic Neo 섞임
3. **Auto Layout 미적용**: 모든 요소가 절대 좌표로 배치 → 내용 변경 시 레이아웃 깨짐
4. **레이어 네이밍 미흡**: "Group", "Rectangle" 등 역할 불명확

---

## 확정된 디자인 룰

- **폰트**: Pretendard 패밀리 통일 (Inter, Apple SD Gothic Neo 사용 금지)
- **Weight 체계**: ExtraBold(800) / Bold(700) / Regular(400)
- **텍스트 관리**: 줄 단위 분리 금지 → 하나의 텍스트 블록으로 관리, 줄바꿈은 텍스트 박스 너비에 따라 자동 처리

---

## 생성된 템플릿 4종

편집 대지 오른쪽 (x: 12000 부근)에 위치.

### A. 커버형 (`TPL-A 커버형`, id: 73:5)
- **용도**: 캐러셀 첫 장, 아티클 제목
- **구조**:
  - 이미지 영역 (Rectangle, 1080×1440, 풀블리드 배경)
  - 타이틀 영역 (Frame, Auto Layout vertical, x:60 y:72, w:960)
    - 타이틀 Line 1~3 (Pretendard ExtraBold 100px, white)
- **기존 예시**: 대지 1 사본 35 ("발렌시아가 스포츠에 진심이다")

### B. 리드텍스트형 (`TPL-B 리드텍스트형`, id: 73:6)
- **용도**: 도입/마무리 문단 장
- **구조**:
  - 프레임 자체가 Auto Layout vertical
  - Padding: top/bottom 72px, left/right 60px
  - Item spacing: 56px
  - 본문 텍스트 (단일 블록, Pretendard ExtraBold 46px, white, w:960)
- **기존 예시**: 대지 1 사본 36 (긴 도입 문단), 대지 1 사본 64, 76, 78

### C. 이미지+캡션형 (`TPL-C 이미지+캡션형`, id: 73:7)
- **용도**: 이미지 배경 위 캡션이 있는 장 (가장 많이 사용되는 유형)
- **구조**:
  - 이미지 영역 (Rectangle, 1080×1440, 풀블리드 배경)
  - 캡션 영역 (Frame, Auto Layout vertical, x:60 y:960, w:960, item spacing 24px)
    - 리드 텍스트 (Pretendard Bold 31px, white, w:960) — 핵심 1~2줄
    - 본문 텍스트 (Pretendard Regular 31px, white, w:960) — 상세 설명
- **기존 예시**: 대지 1 사본 39, 41, 43, 44, 45, 46, 48, 50, 65, 66, 67, 68, 69, 70, 71, 72, 73, 74, 75 등 (대다수)

### D. 이미지전용 (`TPL-D 이미지전용`, id: 73:8)
- **용도**: 풀블리드 이미지, 텍스트 없음
- **구조**:
  - 이미지 영역 (Rectangle, 1080×1440)
- **기존 예시**: 대지 1 사본 38, 40 등

## 남은 작업 (다음 세션)

### 즉시 필요
1. **폰트 일괄 변경**: 템플릿 텍스트가 Inter로 생성됨 → Figma에서 Pretendard로 수동 변경
   - 4개 템플릿 전체 선택 → Cmd+A → 폰트: Pretendard
   - Weight: 레이어명 참고 (ExtraBold 800 / Bold 700 / Regular 400)

2. **타이틀 영역 프레임 배경 투명화**: `TPL-A > 타이틀 영역` (73:10)과 `TPL-C > 캡션 영역` (73:17) 프레임의 fill을 투명 또는 제거

### 선택적 확장
3. **기존 40개 프레임 리팩토링**: 현재 편집 대지의 프레임들을 템플릿 구조로 마이그레이션
   - 이미지 유지하면서 텍스트를 단일 블록으로 병합
   - Auto Layout 캡션 영역으로 교체
4. **Figma 컴포넌트화**: 템플릿을 Component로 등록 → Variant로 A/B/C/D 관리
5. **캡션 위치 변형 추가**: 현재 C형은 하단 캡션만 — 상단 캡션 변형도 필요할 수 있음

## 참고: Figma MCP 제한사항

- `create_text`로 생성 시 폰트 패밀리 지정 불가 (Inter로 고정 생성)

- `export_node_as_image`에서 scale 파라미터 오류 발생
- TEXT 노드에 `set_layout_sizing` 적용 불가
