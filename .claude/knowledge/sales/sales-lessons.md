# 세일즈 레슨런 & 인사이트 축적

> 용도: 매 세일즈 케이스(성공/실패/진행중)의 교훈을 축적하여 시스템 전체의 정확도를 지속적으로 향상
> 사용 에이전트: 모든 세일즈 에이전트가 작업 전 참조

## 축적 원칙

1. **모든 케이스를 기록** — 성공뿐 아니라 실패·무응답도 기록 (실패에서 더 많이 배운다)
2. **구체적으로** — "잘 안 됐다" (X) → "제목에 [섭외 요청]을 넣었더니 열람률이 낮았다. 다음엔 상대 키워드로 시작하는 제목 테스트" (O)
3. **패턴을 발견하면 즉시 에이전트/프레임워크에 반영** — 3번 이상 반복되는 교훈은 해당 에이전트 워크플로우에 통합
4. **분기별 종합 리뷰** — 축적된 레슨런을 분기마다 종합하여 프레임워크 업데이트

## 레슨런 기록 템플릿

새 케이스 종료 시 아래 포맷으로 기록:

```
### [YYMMDD] [대상명] — [채널: 인바운드/아웃바운드/Upwork/스폰서십/섭외]

- **상황**: (어떤 맥락에서 어떤 대상에게 접근했는가)
- **접근 방식**: (어떤 프레임워크/에이전트를 사용했고, 어떤 전략을 썼는가)
- **결과**: 성공 / 실패 / 무응답 / 진행중
- **배운 점**: (1~3개 구체적 교훈)
- **다음에 다르게 할 것**: (개선 방향)
- **적용 대상**: (이 교훈이 적용되는 에이전트/상황)
```

## 에이전트 활용 가이드

모든 세일즈 에이전트는 작업 시작 전:

1. **이 파일의 최근 케이스 확인** — 동일 업종/유형의 레슨런이 있는지 체크
2. **패턴 테이블 확인** — 확인된 패턴이 현재 케이스에 적용되는지 판단
3. **케이스 종료 후 기록 제안** — "이번 케이스의 레슨런을 기록할까요?" 사용자에게 제안
