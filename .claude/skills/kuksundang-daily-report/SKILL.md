# 국순당 데일리 리포트 (2-메시지 포맷)

## Objective
매일 오전에 국순당 프로젝트의 오늘 현황을 **2개의 메시지**로 분리하여 Slack 채널 "outsource-1-국순당"으로 발송한다.

- **메시지 1** — 내부 데일리 현황: 팀 내부 운영 중심
- **메시지 2** — 클라이언트 공유 액션 가이드: 외부 커뮤니케이션 액션 중심

## 실행 순서

### Step 1: Notion DB 조회

**1. 내부 업무관리 (제작 캘린더)**
- Notion data source: `collection://2fd02b0e-30a9-80d6-91c4-000b9f5c2a4b`
- 태그(상태): 예정 / 촬영 / 1차공유 / 2차공유 / 최종공유 / 업로드 / 완료

**2. 2026 국순당 콘텐츠 (마스터 DB)**
- Notion data source: `collection://2e802b0e-30a9-80be-a68b-000b949a80a2`

### Step 2: 3단계 공유 시스템 기반 분석

| 업로드까지 남은 일수 | 해야 할 일 |
|---|---|
| D-14 이내 & 1차공유 전 | 1차 공유(30%) 준비 필요 |
| D-10 이내 & 2차공유 전 | 2차 공유(80%) 준비 필요 |
| D-5 이내 & 최종공유 전 | 최종 공유(100%) 긴급 준비 |
| D-3 이내 & 업로드 전 | 업로드 임박 — 최종 승인 확인 |
| 피드백 마감일 도래 | 클라이언트 피드백 리마인드 필요 |

**내부 vs 클라이언트 단계 매핑:**
- 내부 20% → 클라이언트 "30% 1차 공유"
- 내부 50% → 클라이언트 "80% 2차 공유"
- 내부 80% → 클라이언트 "100% 최종 공유"

### Step 3: 메시지 1 — 내부 데일리 현황 발송

Slack 채널 "outsource-1-국순당" (C0ABALBMYRW)에 발송:

```
[국순당] 데일리 현황 ({YYYY.MM.DD 요일})

오늘 Due
 - {오늘 마감 항목} — {필요 액션}

오전 내부 확인 필요
 - {내부 논의/상태 파악 필요 항목} — {이유}

내일
 - {내일 마감 또는 업로드 항목}

다가오는 일정
 - {콘텐츠명} {액션}: {날짜} (D-N)
```

### Step 4: 메시지 2 — 클라이언트 공유 액션 가이드 발송

```
[국순당] 오늘 클라이언트 공유 액션

1. {콘텐츠명} {공유 단계} (→{완성도}%)
   {날짜} Due — {공유 내용 설명}
   → {담당자}님께: "{피드백 요청 멘트}"
   → 피드백 기한: 공유일 D+2 ({날짜 요일})
```

**공유 단계별 피드백 요청 멘트:**
- 1차 공유(30%): "전체적인 방향성과 흐름에 대해 피드백 부탁드립니다"
- 2차 공유(80%): "디테일적인 부분(자막, 색감, 전환효과 등)에 대해 피드백 부탁드립니다"
- 최종 공유(100%): "최종 확인 후 업로드 진행 예정입니다. 검토 부탁드립니다"
