# 봤어? 스크립트 작성 Skill

## 개요
CH MAG '봤어?' 시리즈 숏폼 스크립트를 기획하고 작성하는 전문 스킬.
사용자가 링크나 주제를 제공하면, A-B-C 프레임워크에 맞춰 완성된 봤어? 스크립트를 생성한다.
스크립트와 함께 **Manychat 공유본** (영상에 못 담긴 풍성한 리서치 정리본)을 자동으로 생성하여, 댓글 전환율을 극대화한다.

## 트리거
MUST BE USED when:
- "봤어 스크립트 써줘", "봤어 새 에피소드", "봤어 기획해줘"
- 링크를 주면서 "봤어로 만들어줘"
- "숏폼 스크립트 써줘" (스포츠/라이프스타일 이슈 맥락)

USE PROACTIVELY when:
- 스포츠 브랜드/테크/라이프스타일 이슈 링크가 공유될 때
- 봤어 시리즈 관련 대화가 시작될 때

## 시리즈 정체성

### 핵심 컨셉
- **제목:** 봤어? (해봤어?)
- **Key Phrase:** "여러분 이거 봤어요?"
- **포맷:** 이슈 소개 숏폼 시리즈 (30~50초)
- **톤:** 친구가 추천하듯 자연스럽고 빠른 톤
- **타겟:** 운동/스포츠/라이프스타일 트렌드에 민감한 20-30대

### A-B-C 프레임워크
- **A (주체):** CH MAG 호스트/큐레이터 — 트렌드를 빠르게 파악하는 사람
- **B (행위):** "이거 봤어?"라는 Key Phrase로 하나의 이슈를 빠르게 소개
- **C (감상):** 호기심, 새로운 발견, 정보 공유의 기쁨 → "오 이건 몰랐네", "친구한테 공유해야지"

## 워크플로우

### Step 1: 소스 분석 + 확장 리서치
사용자가 제공한 링크/주제를 분석한다.

**1-A. 메인 소스 분석:**
- WebFetch로 사용자 제공 링크의 전체 내용 파악
- 핵심 팩트 3~5개 추출
- CH MAG 시청자에게 흥미로운 앵글 도출

**1-B. 확장 리서치 (공유본용):**
- WebSearch로 메인 소스 관련 추가 정보 수집
- 배경 맥락, 관련 인물/브랜드 히스토리, 숫자/데이터 보강
- 영상에서 다루기엔 길지만 알면 재밌는 디테일들 수집
- **목표:** 영상 내용의 2~3배 분량의 풍성한 리서치 확보

**1-C. 시의성 검증 (필수):**
- **최신 정보 우선 원칙:** 리서치 시 가장 최근 정보(현재 연도 기준)를 우선 수집하고, outdated된 내용이 포함되지 않도록 검증한다
- WebSearch 시 현재 연도를 쿼리에 포함하여 최신 결과 확보 (예: "주제 2026")
- 수집한 팩트의 날짜/시점을 반드시 확인 — 오래된 수치나 이미 변경된 정보는 제외하거나 최신 정보로 교체
- 규정, 제도, 순위 등 시간에 따라 바뀌는 정보는 현재 기준으로 업데이트
- **체크:** "이 정보가 지금 시점에도 유효한가?" — 확신이 없으면 추가 검색으로 확인
- 스크립트와 공유본 모두에 outdated 정보가 들어가면 브랜드 신뢰도에 직결되므로, 시의성 검증은 모든 리서치의 마지막 단계로 반드시 수행

**분석 체크리스트:**
```
- [ ] 이슈가 스포츠/라이프스타일과 연결되는가?
- [ ] 시각적으로 보여줄 수 있는 요소가 있는가?
- [ ] "이거 봤어?"라고 물을 만큼 신선한가?
- [ ] 1분 이내에 핵심을 전달할 수 있는가?
- [ ] 공유본에 담을 추가 디테일이 충분한가?
```

### Step 2: 앵글 기획
하나의 핵심 앵글을 설정한다.

**앵글 설정 원칙:**
1. 이슈 자체보다 "왜 이게 중요한지"에 집중
2. 숫자/데이터가 있으면 적극 활용
3. 비교 프레임이 효과적 (기존 vs 혁신, before vs after)
4. 엔딩은 Manychat 공유본으로 전환 유도 (아래 Step 3 참고)

### Step 3: 스크립트 작성

**구조 (4단 구성):**

```
[0–3초 | 훅]
"여러분 이거 봤어요?"
+ (시각적 임팩트 있는 장면 지시)

[3~15초 | 이슈 소개]
- 이게 뭔지 빠르게 설명
- 핵심 팩트 1~2개

[15~30초 | 깊이 있는 포인트]
- 왜 중요한지 / 맥락 설명
- 추가 팩트 2~3개
- 비교 프레임 활용

[마지막 5~8초 | Manychat CTA 엔딩]
- 공유본의 가치를 구체적으로 언급 (뭘 더 알 수 있는지)
- 댓글 행동을 명확하게 지시
- 공유본 받으면 뭐가 좋은지 한 줄로 후킹
```

**Manychat CTA 엔딩 작성 원칙:**

엔딩은 단순 "댓글로 남겨주세요"가 아니라, **공유본의 구체적 가치를 티징**하여 댓글 전환을 유도한다.

**공식:**
> [이 영상에서 다 못 다룬 게 뭔지 티징] + [댓글 행동 지시] + [받으면 뭐가 좋은지]

**좋은 예시:**
- "사실 이 외에도 ~에 대한 이야기가 더 있는데요, 댓글로 '정리본'이라고 남겨주시면 크후가 리서치한 풀버전 정리를 보내드립니다!"
- "오늘 다 다루지 못한 ~의 비하인드, 댓글에 '궁금'이라고 적어주시면 정리본으로 한번 더 보내드릴게요!"
- "여기서 다 보여드리기엔 아까운 디테일이 많거든요. '더보기' 댓글 남겨주시면 크후 리서치팀이 정리한 딥다이브 자료 보내드립니다!"

**나쁜 예시 (이렇게 쓰지 말 것):**
- "여러분 의견이 궁금합니다. 댓글로 남겨주세요!" (전환 동기 없음)
- "좋아요와 구독 부탁드립니다!" (일반적 CTA, 차별화 없음)
- "댓글로 남겨주시면 정리본 보내드립니다!" (뭘 정리했는지 안 알려줌)

**핵심:** 공유본에 실제로 담길 내용 중 가장 흥미로운 것을 콕 집어서 티징해야 한다. 매 에피소드마다 공유본 내용에 맞게 엔딩 CTA를 커스텀한다.

**작성 규칙:**
1. **반말+존댓말 혼합:** "~예요/~거든요" 톤 (친근하지만 정보 전달력 있게)
2. **짧은 문장:** 한 문장이 20자 내외, 길어도 30자 이내
3. **숫자 강조:** 정확한 수치가 있으면 반드시 활용
4. **영상 지시 포함:** 각 구간별 B-roll/영상 연출 힌트 괄호로 표기
5. **Key Phrase 시작:** 항상 "여러분 이거 봤어요?"로 시작
6. **Manychat CTA 엔딩:** 공유본 가치를 구체적으로 티징하여 댓글 전환 유도

### Step 4: 버전 분리
- **풀 버전 (40~50초):** 디테일한 설명 포함
- **축약 버전 (25~30초):** 핵심만 압축

### Step 5: Manychat 공유본 작성

스크립트와 동시에, 댓글 시 Manychat으로 자동 발송될 **리서치 정리본**을 작성한다.

**공유본 구성:**

```markdown
# 봤어? 정리본 — [주제]

> 크후가 리서치한 딥다이브 정리본입니다.
> 영상에서 다 담지 못한 내용까지 풍성하게 준비했어요!

## 영상 핵심 요약
(영상 내용을 3줄로 요약)

## 영상에서 못 다룬 이야기

### 1. [서브토픽 1]
(영상에선 언급만 했거나 아예 빠진 흥미로운 디테일)

### 2. [서브토픽 2]
(배경 맥락, 히스토리, 관련 인물 이야기 등)

### 3. [서브토픽 3]
(숫자/데이터, 비교 분석, 미래 전망 등)

## 크후's Pick — 이건 꼭 알아두세요
(가장 인상적이거나 실용적인 인사이트 1개를 하이라이트)

## 더 궁금하다면
(관련 읽을거리, 영상, 계정 등 큐레이션 2~3개)

---
*크후 (Creatorhood) — 스포츠 라이프스타일의 모든 것*
*@creatorhood.mag*
```

**공유본 작성 원칙:**
1. **영상보다 풍성해야 함:** 영상 내용을 단순 반복하면 안 됨. 영상에 없는 새로운 정보가 최소 60% 이상
2. **읽기 쉬운 구조:** 모바일에서 Manychat으로 받아보는 환경 고려. 짧은 문단, 볼드 키워드, 명확한 소제목
3. **크후 브랜드 톤 유지:** 정보성이되 딱딱하지 않게. 친근하면서도 신뢰감 있게
4. **가치 전달:** 읽고 나서 "댓글 잘 달았다" 느끼게. 공유하고 싶은 퀄리티

### Step 6: 메타 정보 정리

**산출물에 포함할 항목:**
1. 스크립트 (풀 + 축약)
2. Manychat 공유본
3. 키포인트 정리 (숫자 기반 5~6개)
4. 영상 구성 참고 (썸네일 아이디어, B-roll 목록, BGM 톤)
5. Manychat 트리거 키워드 제안 (댓글에서 인식할 키워드 2~3개)
6. 레퍼런스 링크

## 기존 완료 에피소드 (레퍼런스)

### 스타일 레퍼런스
아래는 기존에 작성된 봤어? 스크립트들의 패턴이다. 새 스크립트 작성 시 이 톤과 구조를 참고한다.

**공통 패턴:**
- 항상 "여러분 이거 봤어요?"로 시작
- 중간에 맥락/배경 설명이 자연스럽게 들어감
- 숫자와 구체적 팩트로 신뢰감 형성
- 마무리는 Manychat 공유본 CTA로 댓글 전환 유도

**톤 예시:**
- "축구 유니폼에 ACG 로고가 등장한 건 역대 최초라 그 자체로 화제가 됐었죠."
- "극한 환경을 위한 옷이라는 ACG의 정체성이 북극권 원정이라는 맥락이랑 너무 자연스럽게 맞아떨어지는 거죠."
- "테니스를 연상시키는 건 아무것도 없어요."
- "IOC가 이번 올림픽의 정식 종목으로 채택한 게 지난 2021년이었는데요."

**주제 카테고리:**
- 스포츠 x 패션/브랜드 협업
- 스포츠 테크 혁신
- 올림픽/대회 관련 이슈
- 스포츠 라이프스타일 트렌드

## 산출물 저장

스크립트와 공유본은 다음 형식으로 마크다운 파일로 저장한다:

**스크립트:**
- **파일명:** `봤어_{주제_키워드}_스크립트.md`
- **저장 위치:** 워크스페이스 폴더 (claude-cowork-contents)

**Manychat 공유본:**
- **파일명:** `봤어_{주제_키워드}_공유본.md`
- **저장 위치:** 워크스페이스 폴더 (claude-cowork-contents)

**노션 연동:** 가능하면 Notion '[DB] 2026 CH 영상 콘텐츠 시리즈' DB의 개별콘텐츠로 등록
