### 2.2 CONTROL 라운드 시간 절감 방안

기존 시뮬레이션: 12명 × 8동작 × 2.5분 = 4시간 (순차 진행 시)

**스테이션 분할 방식 (추천)**:

| 스테이션 | 동작 | 동시 진행 인원 | 필요 장비 |
|---|---|---|---|
| A | 오버헤드 스쿼트 + 피스톨 스쿼트 + 터키시 겟업 | 3명 | 바벨 1, 케틀벨 2 |
| B | 핸드스탠드 홀드 + 핸드스탠드 워크 + L-시트 | 3명 | 매트, 측정 테이프 |
| C | 레글리스 로프 클라임 + 링 머슬업 | 2명 (교대) | 로프, 링, 크래시 패드 |

- 스테이션 C는 안전 위험도 높으므로 전담 스포터 + 의무 인력 상시 배치

### 2.3 ACT 1 자존심 배틀 현장 운영 프로토콜

**목적**: 선언 기반 즉석 미니 대결로 라이벌 구도 형성 + 콘텐츠 킬러 장면 확보

**프로토콜**:

| 단계 | 내용 | 시간 | 주의 |
|---|---|---|---|
| 1. 선언 | 전원 모여서 한 명씩 "이거 하나는 자신 있다" 선언 | 15분 | 사전 고지: "선언하면 그 자리에서 증명해야 할 수 있다" |
| 2. 매칭 | 같은 능력 선언자끼리 자동 매칭. MC가 대결 제안 | 5분 | 최대 4~6건으로 제한 (시간 관리) |
| 3. 워밍업 | 미니 대결 참가자 10분 워밍업 시간 부여 | 10분 | **부상 방지 핵심. 생략 금지** |
| 4. 대결 | 미니 버전으로 진행 (아래 종목별 참조) | 매치당 5~8분 | 저지 1명 + 카메라 1대 전담 |
| 5. 리액션 | 즉각 인터뷰 ("진 기분 어때?", "내일 본 라운드에서는?") | 매치당 3분 | 감정이 살아있을 때 바로 확보 |

**미니 대결 종목 (능력별)**:

| 선언 능력 | 미니 대결 | 시간 | 안전 등급 |
|---|---|---|---|
| ENGINE | 비프테스트 레벨 8부터 시작 서바이벌 | 5분 | LOW |
| GRIT | 2분 AMRAP 버피 (누가 더 많이) | 3분 | LOW |
| FORCE | 데드리프트 체중 × 1.5배 시도 (성공/실패) | 5분 | HIGH — 스포터 필수 |
| BLAST | BlazePod 반응 3라운드 (워밍업 대체) | 4분 | LOW |
| REFLEX | 10m 스프린트 2라운드 | 3분 | LOW |
| CONTROL | 핸드스탠드 최대 유지 | 3분 | MEDIUM — 매트 배치 |

**안전 규칙**:
- FORCE 미니 대결(데드리프트)은 반드시 워밍업 세트 2회 선행
- 파워클린은 미니 대결에서 제외 (기술 미숙 위험)
- 의무 인력 상시 대기, 미니 대결 중 통증 호소 시 즉시 중단

### 3.2 스코어링 시스템 설계 — 0~100 변환 기준 제안

**설계 원칙** (프레임워크 3.4 추천 접근 기반):

| 항목 | Season 0 채택안 | 이유 |
|---|---|---|
| 기본 방식 | 참가자 내 백분위 기반 | 소규모(10~16명)에서 절대 기준 설정 불가. 단순 시작 |
| 체중 보정 | 미적용 | 종목별 처방(체중 비율)으로 간접 보정. 직관성 우선 |
| 능력 간 가중치 | 6능력 동일 비중 | 논쟁 회피 + "완성형 운동인" 정의와 일치 |
| 능력 내 복수 종목 | 종목별 점수 → 능력 내 평균 | 종목 수 차이에 무관한 공정성 |

**변환 공식 (12명 기준)**:

```
Step 1: 각 종목 원점수 기록
Step 2: 종목 내 순위 산출 (1위~12위)
Step 3: 백분위 변환 = (N - 순위 + 1) / N × 100
        → 1위 = 100, 2위 = 91.7, 3위 = 83.3, ... 12위 = 8.3
Step 4: 동일 능력 내 복수 종목 평균
        예: FORCE = (데드리프트 100 + 스쿼트 91.7 + 벤치 83.3) / 3 = 91.7
Step 5: 능력별 점수 반올림 (소수점 없이 정수)
Step 6: 종합 점수 = 6능력 평균 (가중치 없음)
```

**실시간 계산 인프라**:

| 도구 | 구현 방식 | 비용 | 리드타임 |
|---|---|---|---|
| **Google Sheets 자동화 (추천)** | 원점수 입력 → 순위/백분위 자동 계산 → Hexagon 차트 자동 생성 (Apps Script) | 무료 | 2주 |
| 전용 웹앱 | Next.js 간단 대시보드 — 입력/계산/시각화 | 개발 시간 | 3~4주 |
| 수기 엑셀 | 최소한의 폴백 | 무료 | 1주 |

**추천**: Google Sheets 자동화를 기본으로, 시각화 출력물(Hexagon SVG)은 디자이너가 템플릿 제작 후 데이터만 연동. D-21까지 완성 + 테스트.

| 박스 → 바벨 스쿼트 (하체→하체) | 60초 | Yes (바벨 무게 세팅) |

**예외 규칙**: 장비 고장/설치 문제 시 전환 시간 최대 120초까지 연장. 저지가 판단. 이 경우 양쪽 선수에게 동일하게 연장.

### 4.5 안전 프로토콜 (1:1 대결 특성 반영)

| 상황 | 프로토콜 | 판정 |
|---|---|---|
| **선수 부상** | 의무 인력 즉시 개입. 경기 중단. 부상 선수가 2분 내 복귀 불가 시 TKO(기술 기권) 처리 | 부상 정도에 따라 해당 이벤트 잔여 매치 출전 금지 가능 |
| **장비 고장** | 경기 중단 → 장비 교체/수리 → 양 선수 동일 조건으로 해당 Phase 재시작 | 저지 판단으로 재시작 지점 결정 |
| **오판 클레임** | 경기 종료 후 5분 이내 클레임 → 영상 리뷰 → 저지 3인 재판정 | 경기 중에는 클레임 불가 (경기 흐름 보호) |
| **기권 의사** | 선수가 "기권"을 말하거나 손을 들면 즉시 경기 종료 | 한번 기권 선언 시 철회 불가 |
| **신체 접촉** | 어떤 Phase에서든 의도적 신체 접촉 = 실격 | 비의도적 접촉은 저지 재량으로 경고 또는 무시 |

### 5.1 QA 리포트 이벤트 관련 합의점 해결

| **1.1 콘텐츠 볼륨 비현실적** | 이벤트 현장에서는 **촬영 효율화에 집중**: (1) 모든 매치에 풀 카메라 투입하지 말고 메인카드 3~4경기만 3카메라, 나머지는 2카메라 (2) 프리뷰/리캡은 현장 촬영 필요 없음 → 편집실에서 제작 → 현장 부담 감소 |

### 5.3 촬영팀과의 협업 체계

**원칙**: 이벤트 운영과 촬영은 **서로의 시간을 먹는 관계**. 이를 인정하고 명시적으로 시간을 배분한다.

**규칙**:

| 규칙 | 내용 | 이유 |
|---|---|---|
| **측정 우선** | 측정 중에는 카메라가 측정에 맞춘다 (카메라 때문에 재측정 안 함) | 공정성 > 촬영 |
| **전환 시간 = 인터뷰 시간** | 라운드 간 전환 20분 중 10분은 인터뷰 촬영에 할당 | 전환 시간을 유휴가 아닌 콘텐츠 생산으로 전환 |
| **카메라 포지션 사전 확정** | D-7 장소 답사 시 라운드별 카메라 위치 확정. 카메라 이동 시간을 전환 시간에 포함 | 현장에서 즉석으로 앵글 찾다가 시간 낭비 방지 |
| **촬영 필수 순간 리스트** | 라운드별 "이 장면은 반드시 잡아야 한다" 리스트 사전 공유 | 편집에서 필수 소스 누락 방지 |
