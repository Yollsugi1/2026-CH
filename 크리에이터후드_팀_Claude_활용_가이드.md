# 크리에이터후드 팀 Claude 활용 가이드

> Git 저장소를 통해 팀이 각자 Claude를 쓰면서도 맥락을 공유하는 방법
> 작성일: 2026-02-24

---

## 1. 왜 Git 저장소를 쓰는가

우리 팀은 각자 Claude를 사용한다. 수현이는 기획/경영, 지운이는 촬영/편집 스케줄, 우석이는 디자인/국순당, 임용이는 에디토리얼.

문제는 **각자 Claude에게 매번 처음부터 설명해야 한다**는 것이다.

- "크리에이터후드는 스포츠 라이프스타일 콘텐츠 스튜디오고..."
- "우리 시리즈는 Hoodies, Playbook, 폭력적인 식단..."
- "국순당 프로젝트는 이런 구조로..."

**Git 저장소에 CLAUDE.md와 스킬/플러그인을 공유하면**, 각자의 Claude가 이미 우리 팀의 맥락을 알고 있는 상태에서 시작한다.

```
팀원 A의 Claude ─┐
팀원 B의 Claude ─┤── Git 저장소 (공유 맥락) ── CLAUDE.md, 스킬, 플러그인
팀원 C의 Claude ─┘
```

## 2. Claude 제품 이해하기

현재 Claude를 쓸 수 있는 방법이 여러 가지인데, 각각 다르다.

### Claude.ai (웹/앱)

| 항목 | 내용 |
|---|---|
| **접속** | claude.ai 웹 또는 Claude 앱 |
| **특징** | 대화형. 파일 첨부 가능. 가장 기본적인 사용법 |
| **한계** | 세션이 끝나면 맥락이 사라짐. 자동화 불가 |
| **적합한 용도** | 단발성 질문, 글 작성, 아이디어 브레인스토밍 |

### Claude Cowork (팀 협업)

| 항목 | 내용 |
|---|---|
| **접속** | Claude 앱 → Cowork 모드 |
| **특징** | **커넥터**(Notion, Slack, Craft 등)를 연결해서 외부 서비스와 연동 |
| **장점** | Plugin/Skill을 설치하면 반복 업무 자동화 가능 |
| **적합한 용도** | 국순당 데일리 리포트, 주간 리포트, 스케줄 관리 |

### Claude Code (개발자/파워유저)

| 항목 | 내용 |
|---|---|
| **접속** | 터미널(Terminal)에서 `claude` 명령어 실행 |
| **특징** | 로컬 파일 시스템에 직접 접근. 코드 작성/실행 가능 |
| **장점** | Git 연동, 파일 생성/수정, 스크립트 실행, 웹사이트 개발 |
| **적합한 용도** | 웹매거진 개발, Zone X 랜딩페이지, 자동화 스크립트, 문서 일괄 관리 |

### 비교 요약

```
Claude.ai     →  대화  →  "물어보고 답 받기"
Claude Cowork →  자동화 →  "Notion/Slack 연결해서 반복 업무 처리"
Claude Code   →  개발  →  "파일 만들고, 코드 짜고, Git으로 관리"
```

**우리 팀 기준**: 대부분의 팀원은 Claude.ai + Cowork 조합이면 충분하다. Claude Code는 수현 대표가 주로 사용하고, 필요 시 확장한다.

## 3. 핵심 개념: Skill, Plugin, Agent

### Skill (스킬)

**"특정 업무를 위한 매뉴얼"**

Claude에게 "이 업무는 이렇게 해"라고 미리 정해둔 지침서다. 한 번 만들면 `/스킬이름`으로 바로 실행할 수 있다.

```
예시: /daily → "오늘 액션아이템 정리해줘"
예시: /weekly → "주간 리포트 만들어줘"
예시: /schedule → "국순당 일정 확인해줘"
```

스킬 안에는 이런 것들이 들어있다:
- 어떤 데이터를 어디서 가져올지 (Notion DB, 로컬 파일 등)
- 어떤 형식으로 출력할지 (보고서, 테이블, Slack 메시지 등)
- 어떤 순서로 처리할지 (워크플로우)

### Plugin (플러그인)

**"여러 스킬을 묶은 패키지"**

하나의 프로젝트에 필요한 스킬들을 하나로 묶은 것이다.

```
예시: kuksundang-workflow.plugin
  ├── /daily        (데일리 리포트)
  ├── /weekly       (주간 리포트)
  ├── /schedule     (일정 관리)
  ├── /client-update (클라이언트 보고)
  └── /plan         (콘텐츠 기획)
```

플러그인을 설치하면 안에 포함된 모든 스킬을 한 번에 사용할 수 있다.

### Agent (에이전트)

**"자율적으로 움직이는 Claude"**

일반적인 Claude는 "시키면 하는" 방식이지만, Agent는 목표를 주면 스스로 판단해서 여러 단계를 수행한다.

```
일반 Claude: "이 파일 읽어줘" → 읽어준다
Agent:       "주간미팅 준비해줘" → 마스터플랜 읽고 → Notion 확인하고
             → 이슈 파악하고 → 브리핑 문서 작성까지 자동으로
```

### CLAUDE.md

**"팀의 공유 메모리"**

프로젝트 루트에 있는 `CLAUDE.md` 파일은 Claude가 세션을 시작할 때 자동으로 읽는다. 여기에 우리 팀의 정보가 다 들어있다:

- 회사 구조, 팀원 역할
- 사업 4대 축
- 콘텐츠 시리즈 7종
- 디렉토리 구조
- 작업 컨벤션

**즉, CLAUDE.md가 있으면 Claude가 "크리에이터후드가 뭐 하는 팀이지?"부터 시작하지 않는다.**

## 4. Git 저장소로 공유되는 것들

Git 저장소(`2026-CH`)를 clone하면, 다음이 자동으로 공유된다:

### 자동 공유 항목

| 항목 | 파일 | 효과 |
|---|---|---|
| **팀 맥락** | `CLAUDE.md` | 모든 팀원의 Claude가 우리 팀을 이해함 |
| **콘텐츠 전략** | `creatorhood_2026_content_strategy.md` | 시리즈 기획 맥락 공유 |
| **콘텐츠 현황** | `CEO_대시보드_콘텐츠_현황_260222.md` | 현재 진행 상황 파악 |
| **국순당 워크플로우** | `국순당-자동화-팀공유/` 폴더 전체 | 플러그인 + 스킬 + 가이드 |
| **CH MAG 스킬** | `claude-cowork-contents/ch-mag-writer.plugin` | 에디토리얼 워크플로우 |
| **Zone X 프로젝트** | `claude-team-zonex/` 폴더 전체 | 기획안, 제안서, 코드 |

### 공유 흐름

```
수현(대표)이 스킬/플러그인을 만든다
         ↓
Git 저장소에 push
         ↓
팀원이 pull (또는 clone)
         ↓
자신의 Claude에 설치
         ↓
바로 사용 가능
```

## 5. 현재 사용 가능한 스킬/플러그인

### 국순당 워크플로우 (`국순당-자동화-팀공유/`)

우석이가 메인 사용자. 설치하면 아래 커맨드를 쓸 수 있다:

| 커맨드 | 기능 | 실행 예시 |
|---|---|---|
| `/daily` | 데일리 리포트 생성 → Slack 자동 발송 | "오늘 현황 보내줘" |
| `/schedule` | 제작 일정 D-N 역산 현황 확인 | "다음 공유 언제?" |
| `/weekly` | 주간 리포트 → Slack 발송 | "주간 리포트 보내줘" |
| `/client-update` | 클라이언트(연정님) 보고 페이지 생성 | "클라이언트 보고 만들어줘" |
| `/plan [주제]` | 콘텐츠 기획안 작성 (A-B-C 프레임워크) | "/plan 순당포착 영동시장" |

**필요 커넥터**: Notion, Slack
**설치 가이드**: `국순당-자동화-팀공유/설치-가이드.md`
**우석 전용 가이드**: `국순당-자동화-팀공유/임우석-Plugin-사용가이드.md`

### CH MAG Writer (`claude-cowork-contents/`)

임용이가 메인 사용자. 에디토리얼 5단계 워크플로우:

| 단계 | 내용 |
|---|---|
| Phase 1 | 소재 접수 & 소싱 확장 |
| Phase 2 | 앵글 & 구조 기획 (반복 수정) |
| Phase 3 | Draft 작성 |
| Phase 4 | 피드백 루프 & 버전 기록 |
| Phase 5 | Instagram 산출물 변환 |

**포함된 레퍼런스**: CH MAG 문체 스타일가이드, 편집 피드백 기록, 원문 글 4편
**Context Starter**: `CH-MAG-WRITER-정임용-Context-Starter.md` — 새 세션에서 복붙하면 이전 작업 이어감

### 콘텐츠 관리 스킬

## 7. 설치 방법 (공통)

### Cowork 사용자 (지운, 우석, 임용)

#### 방법 1: 플러그인 파일 드래그앤드롭
1. Claude Desktop 앱 → Cowork 모드
2. Git에서 받은 `.plugin` 파일을 대화창에 드래그앤드롭
3. 미리보기 → **수락(Accept)** 클릭
4. Notion, Slack 커넥터 연결 확인

#### 방법 2: 스킬 수동 설치
1. `skills/` 폴더의 파일들을 자신의 `.claude/skills/` 경로에 복사
2. Claude 재시작
3. `/스킬이름`으로 테스트

### Git 저장소 받기

```bash
# 처음 한 번만
git clone https://github.com/Yollsugi1/2026-CH.git

# 이후 업데이트 받을 때
cd 2026-CH
git pull
```

```
GitHub (2026-CH) ────────────────────────────────
  │
  ├── CLAUDE.md              ← 모든 Claude가 읽는 팀 맥락
  │
  ├── claude-cowork-contents/

  │   ├── weekly-meeting-prep.skill    ← 주간미팅 준비
  │   └── (콘텐츠 기획 문서들...)
  │
  ├── claude-team-zonex/              ← Zone X 프로젝트
  │
  ├── 국순당-자동화-팀공유/

  │   ├── 설치-가이드.md
```

**핵심**: 수현 대표가 스킬/플러그인을 만들거나 업데이트하면, Git push → 팀원이 pull → 최신 버전 바로 사용. 스킬의 버전 관리와 공유가 Git으로 자동화된다.

## 9. 자주 묻는 질문

### Q. Claude.ai에서도 스킬을 쓸 수 있나?
**아니요.** 스킬/플러그인은 Claude Cowork 또는 Claude Code에서만 사용 가능합니다. claude.ai 웹에서는 대화만 가능합니다.

### Q. 스킬을 안 설치하고 그냥 Claude에게 말하면 안 되나?
됩니다. 하지만 매번 "국순당 프로젝트는 이런 구조고, Notion DB는 여기고, 보고 형식은 이렇고..." 설명해야 합니다. 스킬을 설치하면 이 모든 맥락이 미리 들어가 있어서 "/daily" 한 마디로 끝납니다.

### Q. 다른 팀원이 만든 스킬을 내가 수정해도 되나?
본인 로컬에서는 자유롭게 수정 가능합니다. 다만 Git에 push하면 다른 팀원에게도 영향이 가므로, 수정 사항이 있으면 수현 대표에게 먼저 공유해주세요.

### Q. Git을 안 써본 적 있는데?
`git pull` 한 줄만 기억하면 됩니다. 이것만 하면 최신 스킬/문서가 내 컴퓨터에 동기화됩니다. push(올리기)는 수현 대표가 주로 하고, 팀원은 pull(받기)만 하면 충분합니다.

### Q. CLAUDE.md를 수정하면 어떻게 되나?
`CLAUDE.md`는 팀 전체의 공유 맥락이므로, 수정은 수현 대표가 관리합니다. 추가할 내용이 있으면 말해주세요.
