# MacBook Pro M5 마이그레이션 가이드

## 마이그레이션 전 (현재 맥북에서)

### 1. Git 동기화
```bash
cd ~/Downloads/2026\ CH
git add -A && git commit --no-verify -m "pre-migration: full sync" && git push --no-verify origin main
```

### 2. 그 외
- Migration Assistant 실행해서 새 맥으로 전송
- **사용자 이름은 반드시 `Yollsugi`로 동일하게 설정** (Claude Code 프로젝트 경로가 `/Users/Yollsugi/...` 기준)

---

## 마이그레이션 후 (새 맥북 M5에서)

### Step 1: CLI 도구 한 큐 설치

터미널 열고 아래 복붙:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" && \
eval "$(/opt/homebrew/bin/brew shellenv)" && \
brew install node && \
npm install -g @anthropic-ai/claude-code vercel @googleworkspace/cli && \
echo "✅ 완료! claude --version:" && claude --version
```

> Homebrew 설치 중 비밀번호 입력 요청이 나올 수 있음 — 맥 로그인 비밀번호 입력하면 됨

### Step 2: Homebrew 경로 영구 등록

위 스크립트의 `eval` 은 현재 터미널에서만 유효함. 영구 등록:

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
```

### Step 3: Claude Code 로그인

```bash
claude
# OAuth 로그인 프롬프트 → Claude Max 계정으로 로그인
```

### Step 4: MCP 서비스 재인증

Claude Code 안에서 각 서비스를 처음 사용할 때 재인증 팝업이 뜸:

- [ ] Notion
- [ ] Slack
- [ ] Figma (+ Figma 데스크탑 앱 실행 필요)
- [ ] Gmail

### Step 5: 워크스페이스 확인

```bash
cd ~/Downloads/2026\ CH
git pull
```

### Step 6: VS Code에서 Claude Code 확장 활성화 확인

VS Code 열기 → Extensions → "Claude Code" 검색 → 활성화 상태 확인

## 이관 확인 체크리스트

새 맥에서 터미널에 하나씩 쳐서 에러 없으면 정상:

```bash
ls ~/.claude/settings.json
ls ~/.claude/.credentials.json
ls ~/.claude/plugins/
ls ~/.claude/skills/
ls ~/Downloads/2026\ CH/.claude/skills/
ls ~/Downloads/2026\ CH/.claude/agents/
cat ~/Downloads/2026\ CH/.claude/settings.local.json | head -5
```
