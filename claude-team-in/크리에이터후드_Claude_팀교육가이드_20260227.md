# 크리에이터후드 Claude 팀 교육 가이드

> **작성일**: 2026년 2월 27일
> **대상**: 크리에이터후드 전 팀원 (김지운 PD, 임우석 디자이너 포함)
> **목적**: Claude Chat → Cowork → Code로 확장되는 AI 활용 체계 이해 및 팀 공유 시스템 온보딩

---

## 1. 먼저 알아야 할 것: Chat과 Agent의 차이

우리가 지금까지 써온 Claude는 대부분 **Chat** 모드입니다. 질문하면 답하고, 텍스트를 주면 요약하거나 고쳐주는 식이죠. 이건 Claude의 가장 기본적인 사용 방식인데, 사실 이건 Claude가 할 수 있는 일의 극히 일부입니다.

**Agent(에이전트)** 는 Chat과 근본적으로 다릅니다. Chat이 "한 번 물어보면 한 번 대답"이라면, Agent는 "목표를 주면 스스로 단계를 나누고, 도구를 사용하고, 중간에 확인하면서 끝까지 실행"합니다.

### Chat vs Agent 비교

| 구분 | Chat 모드 | Agent 모드 |
|------|-----------|------------|
| **작동 방식** | 질문 → 답변 (1회) | 목표 → 계획 → 실행 → 확인 (반복) |
| **도구 사용** | 텍스트만 생성 | 파일 생성, 웹 검색, Notion/Slack 연동, 코드 실행 |
| **맥락 유지** | 대화 안에서만 | 프로젝트 전체 맥락 + 외부 시스템 연동 |
| **결과물** | 텍스트 답변 | 실제 파일(docx, pptx, xlsx), Slack 메시지, Notion 페이지 등 |

**쉽게 비유하면**: Chat은 "현명한 조언자"이고, Agent는 "일을 직접 해주는 팀원"입니다.

## 2. Claude의 세 가지 모드 이해하기

### 2-1. Claude Chat (claude.ai 웹/앱)

가장 기본적인 형태입니다. 브라우저나 앱에서 대화하는 방식이죠.

**할 수 있는 것:**
- 텍스트 기반 질문/답변, 요약, 번역, 아이디어 브레인스토밍
- 업로드한 파일 읽기 (PDF, 이미지 등)
- 간단한 코드 생성

**한계:**
- 외부 시스템(Notion, Slack, Google Drive 등)에 직접 접근 불가
- 파일을 직접 만들어서 내 컴퓨터에 저장하는 것이 제한적
- 대화가 끝나면 맥락이 사라짐
- 팀 간 설정이나 지식 공유 불가

---

### 2-2. Claude Cowork (데스크톱 앱 내 Cowork 모드)

Cowork은 Claude 데스크톱 앱 안에서 돌아가는 **에이전트 모드**입니다. 수현님이 현재 주로 사용하는 환경이 바로 이것입니다.

**핵심 차이점:**
- 컴퓨터의 폴더에 직접 접근하여 파일을 읽고, 만들고, 수정할 수 있음
- Notion, Slack, Google Drive 등 외부 서비스와 **커넥터(MCP)**로 연결
- **Skill(스킬)** 을 통해 반복 업무를 자동화
- **Plugin(플러그인)** 으로 스킬을 패키징해 다른 사람과 공유 가능

**비개발자에게 좋은 점:**
- 터미널(검은 화면에 명령어 치는 것)을 몰라도 사용 가능
- 대화하듯이 지시하면 Claude가 알아서 실행
- 격리된 안전한 환경에서 작동하므로 실수로 중요한 파일을 망칠 위험이 낮음

---

### 2-3. Claude Code (터미널 CLI)

개발자를 위한 도구입니다. 터미널에서 직접 Claude를 사용하며, Git 저장소와 깊이 통합됩니다.

**핵심 차이점:**
- 프로젝트의 전체 코드베이스를 맥락으로 이해
- `CLAUDE.md` 파일로 프로젝트별 규칙과 맥락을 정의
- Git과 직접 연동 (커밋, PR, 코드 리뷰 등)
- MCP 서버 설정을 `.mcp.json` 파일로 팀과 공유

**Cowork과의 결정적 차이:**
- Cowork: "내 컴퓨터의 특정 폴더"에서 작업
- Code: "Git 프로젝트" 안에서 작업 — 팀원과 설정을 자연스럽게 공유

## 3. 핵심 개념: Skill, Agent, Plugin, MCP

이 네 가지가 Claude를 단순한 챗봇에서 업무 자동화 시스템으로 바꿔주는 핵심입니다.

### 3-1. Skill (스킬)

**스킬은 "Claude에게 가르친 업무 매뉴얼"입니다.**

예를 들어, "국순당 데일리 리포트 보내줘"라고 말하면 Claude가 알아서:
1. 노션에서 국순당 프로젝트 현황을 읽고
2. 오늘 날짜 기준으로 진행 상태를 정리하고
3. Slack 채널에 리포트 메시지를 보냄

이걸 가능하게 하는 것이 `kuksundang-daily-report`라는 스킬입니다.

**현재 크리에이터후드에 만들어진 스킬 목록:**

| 스킬 이름 | 하는 일 | 트리거 키워드 |
|-----------|---------|---------------|
| **daily-action-items** | 마스터플랜에서 오늘 할 일 추출, 노션 캘린더 DB용 포맷 생성 | "오늘 할 일", "데일리 체크" |
| **weekly-meeting-prep** | 주간미팅 브리핑 자료 자동 생성 | "주간미팅 준비", "미팅 브리핑" |
| **kuksundang-daily-report** | 국순당 데일리 리포트 → Slack 발송 | "국순당 데일리", "데일리 보내줘" |
| **kuksundang-weekly-report** | 국순당 주간 리포트 → Slack 발송 | "국순당 주간 리포트" |
| **kuksundang-schedule-manager** | 국순당 제작 일정 관리, 클라이언트 공유 시점 체크 | "국순당 일정", "마감 확인" |
| **kuksundang-client-update** | 국순당 클라이언트 보고용 주간 업데이트 → Notion 페이지 생성 | "클라이언트 보고" |
| **content-planner** | A-B-C 프레임워크 기반 콘텐츠 기획안 생성 | "콘텐츠 기획", 아이디어 제시 시 |
| **content-frameworks** | 윤성원 A-B-C 프레임워크 참조 | 콘텐츠 시리즈 기획 시 |
| **sales-outreach** | 클라이언트 RFP 분석 → 맞춤 영업 제안서 작성 | "대행영업 스킬 사용해서" |
| **quotation-builder** | 서비스 견적서 자동 생성 | "견적", "가격", "얼마" |
| **brand-knowledge** | 크리에이터후드 역량/사례/전략 참조 DB | 제안서 작성, 영업 시 |
| **proposal-docx** | 크리에이터후드 양식 제안서 Word 문서 생성 | "제안서 만들어줘" |
| **session-backup** | 대화 내용 마크다운 백업 | "백업해줘", "세션 저장" |
| **skill-manager / skill-creator** | 새 스킬 만들기, 기존 스킬 수정 | "스킬 만들어줘" |

### 3-2. Agent (에이전트/서브에이전트)

Agent는 스킬보다 더 자율적입니다. 복잡한 업무를 받으면 내부적으로 여러 개의 서브태스크로 나누고, 각각을 병렬로 처리합니다.

**예시 — "국순당 이번 주 전체 현황 정리해줘"라고 하면:**
- 서브에이전트 1: Notion에서 제작 일정 확인
- 서브에이전트 2: Slack에서 이번 주 국순당 관련 대화 검색
- 서브에이전트 3: 마스터플랜에서 D-day 분석
- → 이 세 개의 결과를 합쳐서 종합 리포트 작성

### 3-3. Plugin (플러그인)

**플러그인은 "스킬을 포장해서 다른 사람에게 건네주는 패키지"입니다.**

스킬은 기본적으로 만든 사람만 쓸 수 있습니다. 하지만 플러그인으로 만들면:
- 팀원 누구나 설치해서 같은 스킬을 사용 가능
- Git 저장소를 통해 버전 관리 가능
- 업데이트하면 팀원 전체에게 자동 반영

**현재 설치된 플러그인:**

| 플러그인 | 설명 |
|----------|------|
| **ch-mag-writer** | CH MAG 매거진 콘텐츠 라이팅 전문 — 소싱 확장 → 앵글 기획 → 구조 설계 → Draft 작성 → 피드백 반영 → Instagram 산출물 변환 |
| **proposal-docx** | 크리에이터후드 양식 제안서 Word 문서 생성 (커버페이지 + 본문 + 테이블 스타일) |
| **cowork-plugin-management** | 플러그인 자체를 만들고 관리하는 도구 |

### 3-4. MCP (Model Context Protocol) — 커넥터

MCP는 Claude가 외부 서비스와 대화할 수 있게 해주는 "다리"입니다.

**현재 연결된 커넥터:**
- **Notion** → 문서 읽기/쓰기, 데이터베이스 조회, 페이지 생성
- **Slack** → 메시지 검색/읽기/보내기, 채널 관리
- **Google Drive** → 파일 검색/읽기
- **Vercel** → 웹사이트 배포/관리
- **Figma** → 디자인 파일 읽기, 코드 변환

## 4. 팀 맥락 공유: 어떻게 같은 "뇌"를 나눠 쓸 수 있는가

### 4-1. CLAUDE.md — 프로젝트의 "사용 설명서"

Git 저장소의 루트에 `CLAUDE.md` 파일을 두면, 그 저장소에서 Claude Code를 사용하는 모든 사람이 **동일한 맥락**으로 Claude를 사용하게 됩니다.

```
프로젝트-루트/
├── CLAUDE.md          ← 프로젝트 전체 규칙 (모든 팀원 공유)
├── .claude/
│   └── CLAUDE.md      ← Git에 올리지 않는 개인 설정
├── .mcp.json          ← MCP 서버 설정 (팀 공유)
└── src/
    └── ...
```

**CLAUDE.md에 넣을 수 있는 것들:**
- 프로젝트 컨벤션 ("제안서는 항상 크리에이터후드 양식을 따른다")
- 브랜드 가이드라인 ("톤은 전문적이면서도 친근하게")
- 자주 쓰는 작업 흐름 ("콘텐츠 기획 시 A-B-C 프레임워크 사용")
- 팀 규칙 ("국순당 관련 작업은 항상 Slack #국순당 채널에 공유")

### 4-2. Git을 통한 스킬/플러그인 공유

```
[수현님이 스킬 생성]
        ↓
[Git 저장소에 커밋 & 푸시]
        ↓
[팀원이 저장소를 클론/풀]
        ↓
[팀원의 Claude에서 동일한 스킬 사용 가능]
```

이 흐름의 핵심은: **스킬을 한 번 만들면 팀 전체가 동일한 품질로 사용할 수 있다**는 것입니다. 수현님이 만든 `kuksundang-daily-report` 스킬을 지운 PD가 가져다 쓸 수 있고, 수현님이 업데이트하면 자동으로 최신 버전을 받을 수 있습니다.

### 4-3. Cowork 사용자와 Code 사용자가 같이 쓰는 법

```
┌─────────────────────────────────────────────┐
│           크리에이터후드 Git 저장소           │
│                                             │
│  CLAUDE.md (팀 공통 맥락)                    │
│  .mcp.json (MCP 서버 설정)                   │
│  .claude/skills/ (공유 스킬)                 │
│  plugins/ (팀 플러그인)                      │
└──────────────┬──────────────────────────────┘
               │
       ┌───────┴───────┐
       │               │
  [Cowork 사용자]  [Code 사용자]
  (임우석 디자이너)  (김지운 PD)
   데스크톱 앱에서     터미널에서
   폴더 연결          git clone
```

- **Cowork 사용자**: 데스크톱 앱에서 해당 폴더를 선택하면 스킬 자동 로드
- **Code 사용자**: 해당 저장소에서 `claude` 명령어를 실행하면 CLAUDE.md + 스킬 자동 로드

## 5. 팀원별 활용 시나리오

### 5-1. 김지운 PD — 콘텐츠 제작 & 프로젝트 관리

지운 PD의 주요 업무: 콘텐츠 기획, 촬영/편집 관리, 클라이언트 커뮤니케이션

**시나리오 A: 국순당 콘텐츠 일정 체크**
```
지운 PD: "국순당 이번주 일정 확인해줘"
Claude: (kuksundang-schedule-manager 스킬 자동 실행)
        → Notion에서 일정 데이터 읽기
        → 오늘 기준 D-day 계산
        → 이번주 마감/공유 시점 정리
        → 결과 출력
```

**시나리오 B: 콘텐츠 기획안 작성**
```
지운 PD: "국순당 신제품 '백세주 제로' 출시 콘텐츠 기획해줘"
Claude: (content-planner + content-frameworks 스킬 연동)
        → A-B-C 프레임워크 적용
        → A(관심 유발): "0칼로리 전통주? 믿기 힘들지만 진짜입니다"
        → B(정보 전달): 제조 과정, 맛 비교, 타겟 분석
        → C(행동 유도): 출시 이벤트 연계
        → 에피소드별 구조 + 캡션 초안까지 생성
```

**시나리오 C: 클라이언트 주간 업데이트**
```
지운 PD: "국순당 이번주 클라이언트 보고 준비해줘"
Claude: (kuksundang-client-update 스킬 실행)
        → 이번주 작업 현황 정리
        → Notion 페이지로 보고서 생성
        → Slack으로 팀 알림
```

**시나리오 D: CH MAG 글 작성**
```
지운 PD: "이 기사 링크 기반으로 CH MAG 스타일 글 써줘" + [링크]
Claude: (ch-mag-writer 플러그인 실행)
        → 소싱 확장 (관련 자료 추가 탐색)
        → 앵글 기획 (스포츠 × 문화 교차점)
        → 구조 설계 (매거진 포맷)
        → Draft 작성
        → 인스타 캐러셀용 텍스트 변환
```

### 5-2. 임우석 디자이너 — 디자인 & 비주얼 제작

우석 디자이너의 주요 업무: 비주얼 디자인, 브랜드 에셋 제작, SNS 이미지 제작

**시나리오 A: 제안서 디자인 자료 준비**
```
우석 디자이너: "펠리카나 제안서 만들어야 하는데, 기본 구조 잡아줘"
Claude: (proposal-docx 플러그인 + brand-knowledge 스킬)
        → 크리에이터후드 제안서 양식 기반 Word 문서 생성
        → 브랜드 역량/사례 자동 삽입
        → 디자인 작업에 필요한 텍스트 콘텐츠 완성
```
→ 우석 디자이너는 이 Word 문서의 구조와 텍스트를 기반으로 디자인 작업에 집중할 수 있음

**시나리오 B: 콘텐츠 캡션과 카피 확인**
```
우석 디자이너: "이번 국순당 카드뉴스 캡션 확인해줘, 길이 맞는지도"
Claude: → 캡션 텍스트 검토
        → 인스타그램 최적 길이 체크
        → 톤앤매너 브랜드 가이드 대비 확인
        → 수정 제안
```

**시나리오 C: Figma 디자인 → 코드 변환 (웹 콘텐츠)**

### Step 2: 기본 사용법 익히기

Chat처럼 자연스럽게 대화하면 됩니다. 다만 몇 가지 팁:

- **구체적으로 말하기**: "도와줘" 보다는 "국순당 이번주 인스타 콘텐츠 캡션 3개 안 만들어줘"
- **파일 첨부 활용**: 참고할 파일이 있으면 드래그&드롭으로 올리기
- **스킬 트리거 사용**: "국순당 데일리 리포트 보내줘"처럼 스킬 이름이나 키워드 사용

Cowork에서 Notion, Slack 등의 커넥터를 연결하면 Claude가 직접 해당 서비스에 접근할 수 있습니다. 설정 방법은 각 커넥터별로 안내가 나옵니다.

### Step 4: 스킬 활용 시작

이미 만들어진 스킬들을 먼저 써보면서 감을 잡으세요. 위의 시나리오들을 직접 시도해보는 것을 권장합니다.

## 7. 자주 묻는 질문 (FAQ)

**Q: Chat에서 하던 것과 뭐가 다른가요?**
A: Chat은 텍스트만 주고받지만, Cowork/Code는 실제로 파일을 만들고, Notion에 글을 쓰고, Slack에 메시지를 보내는 등 "행동"을 합니다.

**Q: 스킬을 꼭 써야 하나요?**
A: 아닙니다. 스킬 없이도 자유롭게 대화하며 작업할 수 있습니다. 스킬은 반복되는 업무의 품질과 속도를 높이기 위한 도구입니다.

**Q: 내가 쓴 내용이 팀원에게 보이나요?**
A: Cowork의 대화 내용 자체는 개인별입니다. 다만 CLAUDE.md나 스킬 파일을 Git으로 공유하면 "Claude가 아는 것"을 팀과 나눌 수 있습니다.

**Q: 실수로 잘못된 작업을 시키면 어떡하나요?**
A: Cowork은 안전한 환경에서 실행되고, 중요한 작업(메시지 전송, 파일 삭제 등) 전에는 항상 확인을 요청합니다. Slack 메시지 발송 같은 작업은 반드시 "보내도 될까요?"라고 물어봅니다.

**Q: Cowork과 Code 중 뭘 써야 하나요?**
A: 터미널이 익숙하지 않다면 Cowork을 권장합니다. 두 환경 모두 같은 스킬과 플러그인을 공유할 수 있으므로, 자신에게 편한 환경을 선택하면 됩니다.

**Q: 스킬을 내가 직접 만들 수도 있나요?**
A: 네. "새로운 스킬 만들어줘"라고 하면 skill-creator가 도와줍니다. 예를 들어 "매번 촬영 체크리스트를 만들어야 하는데, 이걸 스킬로 만들어줘"라고 하면 됩니다.
